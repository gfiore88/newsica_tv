# Refactor Day - 2026-05-28

## Obiettivo
Riorganizzare progressivamente il codice NewsicaTV con approccio incrementale, senza downtime funzionale:
- ridurre duplicazioni;
- separare responsabilita' (application/service layer);
- ridurre i moduli monolitici;
- aumentare testabilita' e stabilita' runtime.

## Problemi strutturali rilevati
1. Logica applicativa duplicata tra runtime (`preparation_agent.py`) e UI/API (`dashboard.py`).
2. Moduli-feature mescolati alla dashboard monolitica.
3. Layer applicativo shorts non centralizzato (prima del refactor odierno).
4. Alcune utility di dominio erano annidate in moduli di presentazione.

## Regole operative adottate
1. Refactor a piccoli passi con commit separati.
2. Ogni passo con test automatici.
3. Nessuna modifica distruttiva dei comportamenti runtime.
4. Migrazione progressiva verso moduli `newsica.*` riusabili da agenti e dashboard.
